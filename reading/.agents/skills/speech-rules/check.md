# 演讲稿质量检查规范 (Quality Checklist)

## 4.1 三级检查体系

### 一级：时间与长度检查 (Time & Length)
- [ ] 总字数是否在594-726字范围内？（660字±10%）
- [ ] 开场是否控制在66字左右（±10%）？
- [ ] 主体是否控制在462字左右（±10%）？
- [ ] 结尾是否控制在132字左右（±10%）？
- [ ] 按正常语速朗读，时长是否在2分42秒-3分18秒之间？

### 二级：内容与逻辑检查 (Content & Logic)
- [ ] 开场是否快速切入主题（前3句话内）？
- [ ] 主体是否包含2-3个清晰要点？
- [ ] 每个要点是否有具体内容支撑（非空泛表述）？
- [ ] 结尾是否回应了开场的主题？
- [ ] 段落间是否有自然过渡？

### 三级：表达与效果检查 (Expression & Effect)
- [ ] 是否有书面语需要口语化？（如"综上所述"→"总的来说"）
- [ ] 是否有技术术语需要通俗解释？
- [ ] 是否有过度谦虚或夸大的表述？
- [ ] 是否有重复或冗余的句子？
- [ ] 演讲时是否有自然的停顿点？

## 4.2 题材特化检查项

| 题材 | 特化检查项 |
|-----|----------|
| 项目汇报 | 价值清晰：听众能否在1分钟内理解项目价值？亮点突出：是否突出了2-3个核心亮点？ |
| 学术报告 | 逻辑严密：研究方法、发现、结论是否逻辑连贯？数据准确：引用的数据是否准确无误？ |
| 毕业答辩 | 成果具体：是否用具体案例展示成果？反思深刻：是否有真实的个人反思？ |

## 4.3 评分与迭代机制
- **基线分**：满分100，一级检查每项10分，二级每项8分，三级每项6分
- **通过标准**：≥85分可输出；70-85分需标注待改进项；<70分重写
- **迭代指令**：若未通过，使用"基于[具体检查项]的反馈，请修正[具体位置]，要求[具体标准]"

## 4.4 常见失败模式 (Anti-patterns)

| 模式名称 | 症状 | 修复策略 |
|---------|------|---------|
| 开场冗长 | 问候、自我介绍用了1分钟 | 强制开场控制在66字内，直接切入主题 |
| 细节过多 | 讲代码实现讲了2分钟 | 删除技术细节，只讲"做了什么"和"有什么价值" |
| 缺乏重点 | 平铺直叙，没有突出亮点 | 用"最让我印象深刻的是..."引出重点 |
| 书面语严重 | "综上所述"、"由此可见" | 全部替换为口语化表达 |
| 结尾仓促 | 最后30秒匆匆结束 | 预留132字给结尾，包含总结、感谢、展望 |

## 4.5 演讲效果自测
- **录音测试**：录下自己的演讲，回听时检查：
  - [ ] 语速是否适中（不快不慢）？
  - [ ] 停顿是否自然？
  - [ ] 语气是否有感染力？
  - [ ] 是否有口头禅（如"那个"、"嗯"）？
- **听众测试**：找1-2位同学试听，询问：
  - [ ] 能否记住项目的核心价值？
  - [ ] 哪些地方听不懂？
  - [ ] 哪些地方印象深刻？
  - [ ] 整体时长是否合适？

## 4.6 最后检查清单
- [ ] 字数检查：660字±10%
- [ ] 时长检查：3分钟±10%
- [ ] 内容检查：开场-主体-结尾完整
- [ ] 表达检查：口语化、无书面语
- [ ] 重点检查：突出2-3个核心亮点
- [ ] 逻辑检查：段落过渡自然
- [ ] 语气检查：自信、清晰、有感染力
