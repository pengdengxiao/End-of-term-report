# 演讲稿主题锚定规范 (Topic Anchoring)

## 1.1 核心主题声明
- **主命题**：在有限时间内（3分钟/660字），向听众清晰传达项目的核心价值与学习成果
- **边界范围**：
  - 必须涵盖：
    - 项目背景与目标（1-2句话）
    - 核心成果与亮点（2-3个关键点）
    - 个人收获与反思（1-2句话）
    - 未来展望（1句话）
  - 明确排除：
    - 技术细节的深度展开（如代码实现、API调用流程）
    - 项目开发过程中的所有困难与挫折
    - 与其他项目的对比分析
- **目标听众**：项目评审老师、同学，需要快速理解项目价值而非技术细节

## 1.2 偏题检测机制
生成过程中，每完成一个段落需自检：
- [ ] 本段是否在660字总预算内？
- [ ] 本段是否直接服务于"让听众理解项目价值"这一目标？
- [ ] 若删除本段，演讲逻辑是否仍然完整？（如否，则本段可能冗余）
- [ ] 本段是否引入了听众不需要知道的技术细节？

## 1.3 题材适配示例
| 题材类型 | 主命题示例 | 边界排除示例 |
|---------|-----------|------------|
| 项目汇报 | 本项目如何通过模块化教学让初学者快速掌握AI Agent开发 | 不讨论具体代码实现、不对比其他教学项目 |
| 学术报告 | 本研究在XX领域的创新点与实际应用价值 | 不罗列所有参考文献、不讨论研究经费来源 |
| 毕业答辩 | 本人在毕业设计中完成的核心工作与能力提升 | 不抱怨时间紧张、不夸大项目难度 |

## 1.4 纠偏触发词
当内容偏离时，使用以下句式强制回归：
"回到核心目标：在3分钟内让听众记住[项目核心价值]，我们聚焦于..."
