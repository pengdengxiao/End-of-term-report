# 如何工作

根据用户需求（演讲时长、语速、主题），创作演讲稿的四个约束规则文件：

| 文件             | 一句话定位      | 解决什么问题          |
| -------------- | ---------- | --------------- |
| `topic.md`     | **讲什么**    | 防止跑题、防止内容过多或过少 |
| `voice.md`     | **怎么讲**    | 设定语气，防止书面语、防止缺乏感染力 |
| `structure.md` | **讲成什么样**  | 防止结构混乱、防止时间分配不合理 |
| `check.md`     | **讲完后查什么** | 防止超时、防止内容空洞、防止表达生硬 |

# 具体规则

## topic.md - 主题锚定规范
- 核心主题声明：在有限时间内传达项目核心价值
- 边界范围：必须涵盖的内容（背景、成果、收获、展望）和明确排除的内容（技术细节、困难挫折）
- 偏题检测机制：每段自检是否在字数预算内、是否服务于核心目标
- 纠偏触发词："回到核心目标..."

## voice.md - 语气与风格规范
- 基础人格设定：自信的项目开发者，分享者-倾听者关系
- 语言风格参数：句式复杂度低-中、修辞密度中、人称第一人称为主
- 禁止事项：书面语、过度谦虚、堆砌形容词、网络流行语
- 演讲节奏控制：220字/分钟、停顿规则、互动设计、呼吸感
- 口语化转换规则：书面语→口语化表达对照表

## structure.md - 结构规范
- 时间分配原则：开场10%、主体70%、结尾20%
- 通用结构骨架：开场→主体（3个要点）→结尾
- 题材定制模板：项目汇报、学术报告、毕业答辩三种模板
- 段落微观结构：每段不超过150字、句式变化、过渡规则
- 长度控制公式：总字数=220字/分钟×3分钟=660字（±10%）
- 重点突出技巧：数字强调、对比强调、设问强调、重复强调

## check.md - 质量检查规范
- 三级检查体系：
  - 一级：时间与长度检查（字数、时长）
  - 二级：内容与逻辑检查（开场、主体、结尾）
  - 三级：表达与效果检查（口语化、术语、语气）
- 题材特化检查项：项目汇报、学术报告、毕业答辩
- 评分与迭代机制：基线分100分，通过标准≥85分
- 常见失败模式：开场冗长、细节过多、缺乏重点、书面语严重、结尾仓促
- 演讲效果自测：录音测试、听众测试
- 最后检查清单：7项核心检查项

# 使用示例

当用户需要撰写演讲稿时：
1. 询问用户：演讲时长、语速、演讲主题、目标听众
2. 根据上述四个规范文件生成演讲稿
3. 使用check.md进行质量检查
4. 若未通过检查，根据反馈迭代修改
