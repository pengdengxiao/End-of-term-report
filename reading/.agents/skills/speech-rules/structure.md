# 演讲稿结构规范 (Structure Blueprint)

## 3.1 时间分配原则（总时长3分钟/660字）
- **开场（10%，约66字）**：吸引注意力，快速切入主题
- **主体（70%，约462字）**：核心内容，分2-3个要点展开
- **结尾（20%，约132字）**：总结升华，留下深刻印象

## 3.2 通用结构骨架
```text
[开场区]（占全文10%，约66字）
  └─ 问候 + 自我介绍 + 演讲主题（一句话概括）
  └─ 示例："大家好，我是XXX。今天我想和大家分享的是..."

[主体区]（占全文70%，约462字）
  └─ 要点1：项目背景与目标（约150字）
     └─ 为什么要做这个项目？解决了什么问题？
  └─ 要点2：核心成果与亮点（约200字）
     └─ 完成了什么？有哪些创新点？
  └─ 要点3：个人收获与反思（约112字）
     └─ 学到了什么？遇到了什么挑战？

[结尾区]（占全文20%，约132字）
  └─ 总结 + 感谢 + 展望
  └─ 示例："总的来说，这个项目让我...感谢大家聆听，谢谢！"
```

## 3.3 题材定制模板

### 模板A：项目汇报演讲
```text
1. 开场（66字）
   - 问候 + 主题引入

2. 项目背景（150字）
   - 为什么要做这个项目？
   - 项目要解决什么问题？

3. 核心成果（200字）
   - 完成了哪些模块？（2-3个核心模块）
   - 有哪些技术亮点？（1-2个关键点）

4. 个人收获（112字）
   - 学到了什么新技能？
   - 有什么深刻体会？

5. 结尾（132字）
   - 总结项目价值 + 感谢 + 未来展望
```

### 模板B：学术报告演讲
```text
1. 开场（66字）
   - 研究背景 + 核心问题

2. 研究方法（150字）
   - 采用了什么方法？
   - 为什么选择这种方法？

3. 核心发现（200字）
   - 主要发现是什么？（2-3个关键发现）
   - 这些发现有什么意义？

4. 研究局限（112字）
   - 研究的不足之处
   - 未来改进方向

5. 结尾（132字）
   - 总结贡献 + 致谢 + 展望
```

### 模板C：毕业答辩演讲
```text
1. 开场（66字）
   - 毕业设计主题 + 个人角色

2. 设计目标（150字）
   - 设计要实现什么功能？
   - 技术难点是什么？

3. 实现成果（200字）
   - 完成了哪些功能模块？
   - 系统有什么特色？

4. 能力提升（112字）
   - 在这个过程中提升了哪些能力？
   - 有什么心得体会？

5. 结尾（132字）
   - 总结 + 感谢导师 + 未来规划
```

## 3.4 段落微观结构
- **每段字数**：不超过150字，超过则拆分
- **句式变化**：避免连续3句同一结构（如都是"我们完成了..."）
- **过渡规则**：段落间必须有显性连接词（"接下来"、"值得一提的是"）

## 3.5 长度控制公式
- 总字数 = 语速 × 时长 = 220字/分钟 × 3分钟 = 660字
- 误差范围：±10%（594-726字）
- 超出660字必须删减，低于594字需补充内容

## 3.6 重点突出技巧
- **数字强调**：关键数据用"整整"、"高达"等词强化
- **对比强调**：用"从...到..."展示变化
- **设问强调**：用"大家可能会问..."引出重点
- **重复强调**：核心观点出现2-3次，每次用不同表述
